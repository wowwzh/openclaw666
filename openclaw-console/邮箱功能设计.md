# OpenClaw Console 邮箱功能设计文档

## 1. 功能概述

### 1.1 需求分析

为 OpenClaw Console 增加邮箱功能，使用户能够：
- 配置邮箱账户（收件）
- 在资讯中心查看收到的邮件列表
- 未配置邮箱时给出友好提示并引导跳转设置

### 1.2 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 邮箱配置 | 在设置页面添加邮箱账户配置 | P0 |
| 收件箱页面 | 在资讯中心添加收件箱Tab | P0 |
| 邮件列表 | 显示最近收到的邮件 | P0 |
| 未开通提示 | 未配置邮箱时显示提示+跳转 | P0 |
| 邮件详情 | 点击查看邮件详细内容 | P1 |

### 3.2 IMAP vs API 对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **IMAP (推荐)** | 通用性强、支持所有邮箱、无需第三方服务 | 需要处理SSL/认证、轮询 | 个人邮箱、自建邮件服务器 |
| Gmail API | 官方支持、可靠 | 需要Google Cloud项目、OAuth配置复杂 | 仅Gmail用户 |
| Outlook API | 官方支持 | 需要Microsoft 365订阅 | 仅Outlook用户 |

**推荐方案：IMAP**

理由：
1. **通用性** - 支持所有主流邮箱服务（Gmail、Outlook、QQ、163等）
2. **免费** - 无需第三方付费服务
3. **简单** - 使用 `imapflow` 库即可轻松连接
4. **自托管** - 无需依赖外部API

### 3.3 推荐技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| IMAP客户端 | [imapflow](https://github.com/postalsys/imapflow) | 高性能Node.js IMAP客户端 |
| 邮件解析 | [mailparser](https://github.com/nodemailer/mailparser) | 解析邮件内容（HTML/文本） |
| 数据存储 | JSON文件 | 简单可靠，无需额外依赖 |
| 定时任务 | node-cron | 定时拉取新邮件 |

### 3.4 数据模型

**邮箱配置 (email_config.json):**
```json
{
  "enabled": true,
  "provider": "gmail",
  "imap": {
    "host": "imap.gmail.com",
    "port": 993,
    "secure": true
  },
  "auth": {
    "user": "user@gmail.com"
    // 密码加密存储
  },
  "sync": {
    "interval": 300000,  // 5分钟
    "lastSync": "2026-02-21T18:30:00Z",
    "folder": "INBOX"
  }
}
```

**邮件缓存 (emails_cache.json):**
```json
{
  "emails": [
    {
      "id": "unique-message-id",
      "subject": "邮件主题",
      "from": {
        "name": "发件人名称",
        "address": "sender@example.com"
      },
      "to": ["user@example.com"],
      "date": "2026-02-21T18:30:00Z",
      "preview": "邮件预览内容...",
      "hasAttachments": true,
      "isRead": false,
      "isStarred": false
    }
  ],
  "totalCount": 156,
  "lastUpdated": "2026-02-21T18:30:00Z"
}
```

### 3.5 API 设计

**Gateway 邮箱相关 API：**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/email/config` | 保存邮箱配置 |
| GET | `/api/email/config` | 获取邮箱配置 |
| POST | `/api/email/test` | 测试邮箱连接 |
| GET | `/api/email/list` | 获取邮件列表 |
| GET | `/api/email/:id` | 获取邮件详情 |
| POST | `/api/email/sync` | 手动触发同步 |
| DELETE | `/api/email/config` | 删除邮箱配置 |

### 3.6 安全考虑

1. **密码加密** - 使用 AES 加密存储邮箱密码
2. **应用专用密码** - 提示用户使用邮箱的应用专用密码（非登录密码）
3. **OAuth 备选** - 后续可增加 Gmail/Outlook OAuth 支持
4. **错误处理** - 连接失败时友好提示（可能是密码错误、IMAP未开启等）

## 5. 存储扩展

需要扩展 `store.ts` 中的 Settings 接口：

```typescript
interface EmailConfig {
  enabled: boolean
  provider: string
  imap: {
    host: string
    port: number
    secure: boolean
  }
  auth: {
    user: string
  }
  sync: {
    interval: number
    lastSync: string | null
    folder: string
  }
}

interface Email {
  id: string
  subject: string
  from: { name: string; address: string }
  to: string[]
  date: string
  preview: string
  hasAttachments: boolean
  isRead: boolean
  isStarred: boolean
}

interface Settings {
  // ... 现有字段
  email: EmailConfig | null
}

// AppState 增加
emails: Email[]
setEmails: (emails: Email[]) => void
fetchEmails: () => Promise<void>
```
