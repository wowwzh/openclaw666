## 2. 性能问题

### 2.1 useMemo 依赖数组问题

**位置**: `Dashboard.tsx`

```tsx
const stats = useMemo(() => [
  { label: '网关状态', value: gatewayStatus.online ? '🟢 在线' : '🔴 离线', ... },
  // ...
], [gatewayStatus, tasks])
```

**问题**: `gatewayStatus` 对象每次渲染都是新引用，导致 useMemo 可能不会正确缓存。

**建议**: 将 gatewayStatus 的具体属性作为依赖，而不是整个对象。

### 2.2 未使用 React.memo

**严重程度**: 中

所有页面组件和 Sidebar 都没有使用 `React.memo` 包装，在父组件重渲染时会不必要地重渲染。

**建议**: 对纯展示组件使用 memo 优化。

### 2.3 内联函数和对象

**严重程度**: 低

多个组件中使用大量内联函数和内联对象：

```tsx
// Dashboard.tsx
{stats.map((stat, i) => (
  <div key={i} ...>...</div>
))}
```

每次渲染都会创建新的函数和对象。

## 4. 潜在的 Bug

### 4.1 Settings 页面复选框逻辑错误 🚨

**位置**: `Settings.tsx` - 消息通道部分

```tsx
// 当前代码 - 所有渠道都使用同一个复选框状态
{[
  { key: 'feishu', name: '飞书', ... },
  { key: 'telegram', name: 'Telegram', ... },
  { key: 'discord', name: 'Discord', ... },
].map((ch) => (
  <input 
    type="checkbox" 
    checked={settings.channels.feishu}  // ❌ 错误：始终使用 feishu
    onChange={(e) => updateSettings({ 
      channels: { ...settings.channels, feishu: e.target.checked }  // ❌ 错误
    })} 
  />
))}
```

**问题**: 
1. 所有三个渠道的复选框都绑定到 `feishu` 状态
2. 无论点击哪个复选框，都只修改 feishu 的值

**修复建议**:
```tsx
checked={settings.channels[ch.key as keyof typeof settings.channels]}
onChange={(e) => updateSettings({ 
  channels: { ...settings.channels, [ch.key]: e.target.checked }
})}
```

### 4.2 Chat 页面使用 eval() 风险 ⚠️

**位置**: `CommandPanel.tsx`

```tsx
const calculate = () => {
  try {
    setResult(eval(calcInput).toString())  // ❌ 安全风险
  } catch {
    setResult('计算错误')
  }
}
```

**问题**: 使用 `eval()` 执行用户输入有严重的安全风险。

**修复建议**: 使用专门的数学表达式解析库，如 `mathjs`。

### 4.3 Tasks 页面使用 confirm() ⚠️

**位置**: `Tasks.tsx`

```tsx
const handleDelete = (id: string) => {
  if (confirm('确定要删除这个任务吗?')) {  // ❌ 不一致的 UI
    deleteTask(id)
    toast.success('任务已删除')
  }
}
```

**问题**: 使用浏览器原生 confirm() 对话框，与整体 UI 风格不一致。

**修复建议**: 使用自定义的确认对话框组件。

### 4.4 Chat 输入框 Enter 键处理

**位置**: `Chat.tsx`

```tsx
onKeyDown={(e) => e.key === 'Enter' && !e.shiftKey && handleSend()}
```

**问题**: 当输入框为空时也会触发发送逻辑。

**建议**: 添加输入验证。

## 5. 其他建议

