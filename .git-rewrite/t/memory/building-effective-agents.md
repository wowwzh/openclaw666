# Anthropic《Building Effective Agents》学习笔记

## 核心观点

**最成功的实现不是使用复杂框架，而是使用简单、可组合的模式。**

---

## 关键概念区分

| 概念 | 定义 |
|------|------|
| **Workflows** | 通过预定义代码路径编排LLM和工具 |
| **Agents** | LLMs动态指导自己的流程和工具使用 |

---

## 什么时候用Agent？

- ✅ 灵活性和模型驱动决策时
- ✅ 复杂任务需要扩展性时
- ❌ 简单任务不需要Agent

**建议**：从最简单的解决方案开始，只在需要时才增加复杂性。

---

## 框架建议

| 推荐 | 不推荐 |
|------|--------|
| 直接使用LLM API | 盲目添加框架 |
| 理解底层代码 | 被框架抽象隐藏 |

**建议**：许多模式可以用几行代码实现。

---

## 构建模块

### 基础：Augmented LLM
增强的LLM = LLM + 检索 + 工具 + 记忆

**关键实现要点**：
1. 为特定用例定制这些能力
2. 确保提供简单、文档良好的接口

---

## 常见模式（从简单到复杂）

1. **Prompt Engineering** - 简单提示
2. **RAG** - 检索增强
3. **Tool Use** - 工具使用
4. **Compositional Workflows** - 组合工作流
5. **Autonomous Agents** - 自主Agent

---

## 我们的启示

1. ✅ 不要盲目用框架，直接用API
2. ✅ 从简单开始，逐步增加复杂度
3. ✅ 关注核心：Prompt + RAG + Tool Use
4. ✅ 保持可组合性和简洁性
