## React 性能优化 (useMemo/useCallback)

### useMemo
- **作用**: 缓存计算结果，避免每次渲染都重复计算
- **适用场景**: 数据转换、复杂计算等昂贵操作
- **示例**:
```javascript
const computedData = useMemo(() => {
  return computeSomething(data);
}, [data]); // 依赖项
```

### useCallback
- **作用**: 缓存函数引用，避免子组件不必要的重新渲染
- **适用场景**: 父组件向子组件传递函数时
- **示例**:
```javascript
const handleClick = useCallback(() => {
  doSomething();
}, [依赖项]);
```

### 性能优化技巧
1. **合理设置依赖项** - 只包含必要的依赖
2. **避免过度使用** - 不要滥用，只有确实有性能问题时才使用
3. **结合 React.memo** - 对组件进行包装
4. **使用 Profiler 工具** - 定位真正需要优化的组件

### 注意事项
- 过度使用会增加内存开销
- 只有当组件确实出现性能问题时才考虑使用
- 大多数情况下 React 的重渲染成本并不高

## Zustand 状态管理

### 简介
Zustand 是一个轻量级的 React 状态管理库，API 简洁，体积小。

### 核心特点
- 简单易用
- 无 Provider 嵌套
- 支持 middleware
- TypeScript 友好

### 基本用法
```javascript
import { create } from 'zustand'

const useStore = create((set) => ({
  count: 0,
  increment: () => set((state) => ({ count: state.count + 1 })),
}))

// 使用
function Counter() {
  const { count, increment } = useStore()
  return (
    <div>
      <span>{count}</span>
      <button onClick={increment}>+</button>
    </div>
  )
}
```
