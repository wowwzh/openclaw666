> 创建时间：2026-02-22

## 一、代码审查 Checklist

### 1. 代码规范
- [ ] 变量/函数命名清晰
- [ ] 代码格式化 (Prettier)
- [ ] 无硬编码 Magic Number
- [ ] 注释适度，不过多不过少

### 2. 性能
- [ ] 无不必要的重渲染 (React)
- [ ] 避免内存泄漏
- [ ] 合理使用 useMemo/useCallback
- [ ] 图片/资源优化

### 3. 安全
- [ ] 无敏感信息泄露
- [ ] 输入验证
- [ ] XSS防护
- [ ] 权限检查

### 4. 错误处理
- [ ] 异常捕获
- [ ] 用户友好的错误提示
- [ ] 日志记录

### 5. 测试
- [ ] 关键逻辑有测试
- [ ] 测试覆盖核心功能

## 二、Git 提交规范

### 提交信息格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型
- feat: 新功能
- fix: Bug修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 构建

### 示例
```
feat(auth): 添加登录验证码

新增短信验证码登录方式
- 发送验证码API
- 验证接口
- 前端表单

Closes #123
```

## 三、PR流程

1. **创建分支**: `git checkout -b feature/xxx`
2. **开发**: 编写代码，添加测试
3. **自检**: 对照Checklist检查
4. **提交**: `git commit -m "feat: xxx"`
5. **推送**: `git push origin feature/xxx`
6. **PR**: 描述改动，关联Issue
7. **Review**: 等待审查，修改反馈
8. **合并**: Review通过后合并

---

## 四、审查评分

| 评分 | 说明 |
|------|------|
| ✅ Pass | 可以合并 |
| 💬 Comment | 建议修改 |
| 🔧 Request Changes | 必须修改 |

---

## 五、常见问题

| 问题 | 建议 |
|------|------|
| 提交信息不清楚 | 按规范写清改动内容 |
| 大PR难review | 拆分成小PR |
| 忘记测试 | 提交前强制自检 |
| 风格不统一 | 配置Prettier自动格式化 |
