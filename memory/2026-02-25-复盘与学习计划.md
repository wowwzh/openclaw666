## 📋 今日难题复盘 (2026-02-25)

### 难题1: OpenClaw Console Gateway 连接失败

**问题描述**:
- WebSocket 连接 Gateway 一直报 `device identity mismatch` 错误
- 即使生成新的设备ID和密钥对，Gateway 仍然拒绝连接

**尝试的解决方案**:
1. ❌ 清除浏览器 localStorage - 失败
2. ❌ 在 Gateway paired.json 中删除旧设备 - 失败
3. ❌ 使用 dangerouslyDisableDeviceAuth 配置 - 失败
4. ✅ 成功方案：跳过 ECDSA 设备认证，直接使用 token 认证

**关键发现**:
- Gateway 的 WebSocket 连接支持两种认证方式：ECDSA 设备认证 + token 认证
- 旧设备认证方案需要私钥签名，但浏览器刷新后私钥丢失
- 使用已注册的 token 可以直接认证成功

**学到的知识**:
- Gateway WebSocket 认证机制
- ECDSA P-256 密钥对生成
- WebSocket 实时事件处理 (health, tick, presence)
- 数据缓存策略
