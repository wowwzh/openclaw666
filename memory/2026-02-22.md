## 05:48 自主练习 - Zustand状态管理学习

### 项目store.ts分析

项目使用了 Zustand + persist 中间件，实现了一个完整的状态管理系统。

#### 核心结构
```typescript
export const useAppStore = create<AppState>()(
  persist(
    (set) => ({
      // 状态定义
    }),
    { name: 'openclaw-storage' }  // 持久化到 localStorage
  )
)
```

#### 包含的状态模块
1. **加载状态** - isLoading (gatewayStatus, tasks, chat)
2. **错误状态** - error
3. **UI状态** - sidebarCollapsed, currentPage
4. **Gateway状态** - gatewayStatus
5. **任务** - tasks (CRUD + API同步)
6. **技能** - skills
7. **设置** - settings
8. **消息/对话** - messages, conversationId
9. **日志** - logs
10. **通知** - notifications

#### 值得学习的点
1. **异步操作模式** - fetchTasksFromApi 中处理 loading/error 状态
2. **状态更新模式** - 使用展开运算符 `...state` 保持不可变性
3. **持久化配置** - persist 中间件自动保存到 localStorage
4. **类型定义完善** - 每个实体都有完整接口

#### 可优化点
1. 可以使用 **Zustand middleware** 简化异步逻辑 (devtools, persist)
2. **selectors** 未充分使用 - 可以提取专门选择器减少重渲染
3. 可以拆分 store 为多个小 store (tasksStore, settingsStore)

### 相关文件
- `openclaw-console/src/store.ts` - 主状态管理
- `openclaw-console/src/api/gateway.ts` - API调用
