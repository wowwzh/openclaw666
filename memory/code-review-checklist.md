# 前端代码审查清单

> 适用于 HTML/CSS/JavaScript 代码审查

## 一、HTML 审查清单

### 1.1 语义化 & 可访问性
- [ ] 使用语义化标签（`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`）
- [ ] 正确使用标题层级（`h1` → `h6`，不要跳过层级）
- [ ] 图片添加 `alt` 属性（装饰性图片用 `alt=""`）
- [ ] 按钮和链接有明确的文字描述（避免"点击这里"）
- [ ] 表单控件关联 `<label>` 标签
- [ ] 支持键盘导航（focus 顺序合理）

### 1.2 性能
- [ ] 减少 DOM 嵌套层级
- [ ] 避免内联大量 CSS/JS
- [ ] 首屏关键 CSS 内联，其余异步加载

### 1.3 代码质量
- [ ] DOCTYPE 声明正确
- [ ] `lang` 属性设置正确
- [ ] meta viewport 正确配置
- [ ] title 和 meta description 完整
- [ ] HTML 缩进一致

---

## 二、CSS 审查清单

### 2.1 响应式设计
- [ ] 使用相对单位（rem/em/百分比）而非固定 px
- [ ] 移动端布局使用 Flexbox 或 Grid
- [ ] 媒体查询断点合理

### 2.2 选择器优化
- [ ] 避免使用 ID 选择器（优先级过高）
- [ ] 避免过长的选择器链
- [ ] 使用 BEM 或类似命名规范
- [ ] 类名语义化、清晰

### 2.3 性能
- [ ] 避免使用 `@import`
- [ ] 合并 CSS 文件（减少 HTTP 请求）
- [ ] 移除未使用的 CSS
- [ ] 动画使用 `transform` 和 `opacity`

### 2.4 可维护性
- [ ] CSS 变量（自定义属性）用于主题色
- [ ] 通用样式抽离为公共类
- [ ] 注释清晰，分块组织
- [ ] 遵循项目约定的 CSS 命名规范

### 2.5 Flexbox & Grid
- [ ] flex 子项明确 `flex-shrink` 策略（防止压缩）
- [ ] grid 布局有合理的回退方案

---

## 三、JavaScript 审查清单

### 3.1 代码风格
- [ ] 使用 ESLint 并遵循项目配置
- [ ] 变量/函数命名有意义（camelCase）
- [ ] 常量使用大写下划线（UPPER_SNAKE_CASE）
- [ ] 缩进和格式一致

### 3.2 变量与类型
- [ ] 优先使用 `const`，其次 `let`，避免 `var`
- [ ] 不使用隐式类型转换（=== vs ==）
- [ ] 字符串拼接使用模板字符串
- [ ] 解构赋值合理使用

### 3.3 函数
- [ ] 函数职责单一（SRP）
- [ ] 参数过多时使用对象参数
- [ ] 使用箭头函数避免 `this` 问题
- [ ] async/await 正确处理错误

### 3.4 DOM 操作
- [ ] 缓存 DOM 查询结果
- [ ] 批量更新使用 DocumentFragment
- [ ] 事件委托代替多个监听器
- [ ] 移除不再使用的事件监听器

### 3.5 性能
- [ ] 避免在循环中同步修改 DOM
- [ ] 使用防抖（debounce）/ 节流（throttle）
- [ ] 大数据列表使用虚拟滚动
- [ ] 图片懒加载

### 3.6 错误处理
- [ ] try-catch 包裹可能抛错的代码
- [ ] 异步操作有错误回调或 catch
- [ ] console.log 替换为正式日志（生产环境）

### 3.7 安全
- [ ] 用户输入进行转义（防止 XSS）
- [ ] 不使用 `eval()`
- [ ] 敏感数据不存放在前端

---

## 四、React/Vue 特定（如适用）

### 4.1 React
- [ ] 使用函数组件 + Hooks
- [ ] 依赖数组完整（useEffect/useMemo/useCallback）
- [ ] key 使用稳定唯一 ID
- [ ] 避免内联函数定义（影响渲染）
- [ ] 状态提升合理

### 4.2 Vue
- [ ] 使用 Composition API
- [ ] v-for 必须带 key
- [ ] computed 属性不含副作用
- [ ] 组件props使用类型定义

## 五、通用审查项

### 5.1 Git 提交规范
- [ ] 提交信息清晰（feat/fix/docs style 等）
- [ ] 不提交敏感信息（API Key、密码）
- [ ] 不提交 node_modules、dist

### 5.2 测试
- [ ] 关键业务逻辑有单元测试
- [ ] 组件有基本的渲染测试

### 5.3 文档
- [ ] 复杂逻辑有注释说明
- [ ] 公共组件有使用文档

---

## 快速检查流程

1. **跑一遍 ESLint** → 格式问题自动修复
2. **检查 HTML 语义化** → 可访问性基础
3. **检查 CSS 变量和响应式** → 维护性和适配
4. **检查 JS 性能和错误处理** → 运行时稳定
5. **检查安全风险** → XSS、敏感信息
